용어정의 : 위키백과 검색.
프로토콜 
- 컴퓨터들 간의 원활한 통신을 위해 지키기로 약속한 규약.
- 프로토콜에는 신호 처리법, 오류처리, 암호, 인증, 주소 등을 포함한다.

OSI참조모델 : 프로토콜을 7계층으로 분류
- 컴퓨터 네트워크 포로토콜 디자인과 통신을 계층으로 나누어 설명한 것.
계층 1 : 물리계층(Physical layer)
계층 2 : 데이터 링크 계층(Data link layer)
계층 3 : 네트워크 계층(Network layer)
계층 4 : 전송 계층(Transport layer)
계층 5 : 세션 계층(Session layer)
계층 6 : 표현 계층(Presentation layer)
계층 7 : 응용 계층(Application layer)

TCP/IP모델 : 프로토콜을 4계층으로 분류	
인터넷 프로토콜 스택의 계층 구조.
계층 1 : 네트워크 접근 계층
계층 2 : 인터넷 계층
계층 3 : 전송계층
게층 4 : 응용계층

LAN(Local Area Network) - 네트워크 인퍼테이스 계층
- 근거리 통신망
- 일반적으로 하나의 사무실 또는 주택 내의 네트워크를 LAN이라고 부른다.
- 광역 통신망에 다이렉트로 연결되는 단말이나, 도시 각 구역에 존재하는 도시권 통신망을 통해 통신을 하는 
  가장 하위층 통신망이며, 통신망의 대부분을 차지한다.	
- 근거리 통신망을 구축하기 위해선 랜 카드와 랜 케이블을 필요로 함.

WAN(Wide Area Network)
- 광역 통신망
- 가장 상위 단위에 속하는 통신망.
- 전국에 깔려있는 전화선,  ISP가 인터넷 서비를 제공하기 위해 전국에 회선을 깔아 구축한 통신망.
- 인터넷 회선뿐 아니라 최상위 단위의 네트워크끼리 다이렉트로 연결되는 경우도 광역통신망으로 불린다
  대표적으로 기업의 인트라넷에서 본사와 지사 간 전용선으로 다이렉트로 연결되는 경우.		

MAC(Media Access Control)/매체 접근 제어
- 48비트로 구성되어 있는 주소.
- MAC주소는 네트워크 세그먼트의 데이터 링크 계층에서 통신을 위한 네트워크 인터페이스에 할당된 고유 식별자이다.
- LAN에서 하나의 통신 회선을 여러 단말장치들이 원할하게 공유할 수 있도록 해주는 통신 회선에 대한 접근 방식 등이 있다.		

NIC(Network Interface Card) : 네트워크 어댑터	
- PC나 서버 등의 컴퓨터를 네트워크에 연결시키기 위한 장치.
- 네트워크 카드, 이더넷 카드, LAN 어댑터, LAN 카드 라고도 한다.
	
이더넷(Ethernet)
- 컴퓨터 네트워크 기술의 하나로, 일반적으로 LAN, MAN 및 WAN에서 가장 많이 활용되는 기술 규격이다.
- 인터넷을 하기위한 수단중 하나로 보면 되고 유선방식으로 이용한다. 
	
패킷	
- 패킷 방식 컴퓨터 네트워크로 수행하는 데이터의 서식 단위.
- 통신망을 통해 전송하기 쉽도록 자른 데이터의 전송 단위.
	
IP(Internet Protocol) -  TCP/IP 인터넷 계층에 위치.
- 네트워크 상에서 사용되는 각종 장비들(라우터,서버,PC)등을 구분하고 통신하기 위해 부여되는 논리적인 주소.
- 예를 들어 192.168.43.100 이라는 IP주소가 있을 때 192.168. 까지를 네트워크 부분이라 하고 뒤에 .43.100을 호스트 부분이라고 한다.	
- 네트워크 부분이 같아야 장비간 통신이 가능하고 호스트 부분은 반드시 서로 달라야 한다.

IP주소의 사설주소 범위
- 공인 IP와 사설 IP 두가지 종류가 있다. 
	ㄴ 공인 IP는 인터넷에서 정식으로 할당된 IP주소를 뜻한다.
	ㄴ 사설 IP는 LAN영역에서 공인 IP의 주소 부족 문제를 해결 하기 위한 하나의 방편이며, LAN 에서만 사용가능하다
	    또한 LAN에서 사설 IP를 사용 할 경우에도 인터넷 접속을 하기 위해서는 사설 IP를 공인 IP로 변환 해 줄 수 있는
  	    라우터 또는 프락시서버 같은 장비가 필요하다.
- 인터넷에서만 사용되는 전화번호라고 생각할 수 있다.

라우터 
- 논리적으로 분리된 망, 혹은 물리적으로 분리된 망 사이를 지나가야 하는 패킷들에게 경로를 뽑아 자기가 아는
  최상의 경로를 찾아낸 뒤 다른 망으로 패킷을 보내주는 역할을 하는 기계.
- 패킷의 위치를 추출하여, 그 위치에 대한 최적의 경로를 지정하며 이 경로를 따라 데이터 패킷을
  다음 장치로 전달한다.
- 서로 다른 네트워크 간에 중계 역할을 해주는 장치다.
	
TCP(Transmission Control Protocol) 전송제어프로토콜 - 전송계층에 위치
- 근거리 통신망이나 인트라넷, 인터넷에 연결된 컴퓨터에 실행되는 프로그램 간에 일련의 옥텟을 안정적으로,
  순서대로, 에러없이 교환할 수 있게 한다.	

http(HyperText Transfer Protocol)
- 웹 상에서 정보를 주고 받을수 있는 프로토콜이다.
- 주로 TCP를 사용하고 HTML 문서를 주고받는 데에 쓰인다.
- 클라이언트와 서버 사이에 이루어지는 요청/응답 프로토콜이다.
		
https(HyperText Transfer Protocol Secure Socket Layer)
- HTTP의 보안이 강화된 버전이다. 
- 소켓 통신에서 일반 텍스트를 이용하는 대신에, SSL이나 TLS 프로토콜을 통해 세션 데이터를 암호화한다. 따라서
  데이터의 적절한 보호를 보장한다.
		
DNS(Domain Name System)
- 호스트의 도메인 이름을 호스트의 네트워크 주소로 바꾸거나 그 반대의 변환을 수행할 수 있도록 하기 위해 개발되었다.
- 특정 컴퓨터의 주소를 찾기위해, 사람이 이해하기 쉬운 도메인 이름을 숫자로 된 식별 번호(IP 주소)로 변환해 준다.		

DHCP(Dynamic Host Configuration Protocol) 동작 호스트 구성 프로토콜
- 호스트 IP 구성 관리를 단순화하는 IP표준이다. 동적 호스트 구성 프로토콜 표준에서는 DHCP 서버를 사용하여 IP 
  주소 및 관련되 기타 구성 세부 정보를 네트워크 DHCP 사용 클라이언트에게 동적으로 할당하는 방법을 제공한다.
- 인터페이스 및 호스트에 IP 주소를 자동으로 할당해서 사용할수 있도록 해주는 것.

----------------------------------------------------------------------------------------------------------------------------------------------------------------
1) MAC(Media Access Control) 주소

:: 데이터 링크계층에서 NIC 를 구별하기 위해 사용되는 식별값이다. 6바이트 주소로 3바이트는 벤더의 고유값이 사용된다.

2) NIC(Network Interface Controller)

:: 컴퓨터를 네트워크에 연결하여 통신하기 위해 사용하는 하드웨어장치이다. 랜카드 또는 이더넷 카드라고도 한다.

3) LAN

:: 근거리 통신망 Local Area Network 의 약자이다. 네트워크 매체를 이용하여 집, 사무실, 학교 등의 건물과 같은 가까운 지역을 한데 묶은 네트워크이다.

4) WAN

:: 광역 통신망 Wide Area Network 의 약자이다.

5) 이더넷

:: 데이터 링크계층에서 MAC(media access control) 전송방식과 프로토콜의 형식을 정의한다. 이더넷 기술은 대부분 IEEE 802.3 규약으로 표준화되었다.

6) IP

:: Internet Protocol. OSI 네트워크 계층에서 호스트의 주소지정과 패킷 분할 및 조립 기능을 담당한다. 이 때 사용되는 호스트의 식별주소가 IP주소이다.

7) 라우터

::  컴퓨터 네트워크 간에 데이터 패킷을 전송하는 네트워크 장치다. 패킷의 위치를 추출하여,

그 위치에 대한 최적의 경로를 지정하며, 이 경로를 따라 데이터 패킷을 다음 장치로 전달한다. 네트워크 계층에서 사용되는 장치이다.

8) TCP

:: TCP는 근거리 통신망이나 인트라넷, 인터넷에 연결된 컴퓨터에서 실행되는 프로그램 간에 일련의 옥텟을안정적으로,

순서대로, 에러없이 교환할 수 있게 한다. TCP는 전송 계층에 위치한다.

TCP는 웹 브라우저들이 월드 와이드 웹에서 서버에 연결할 때 사용되며, 이메일 전송이나 파일 전송에도 사용된다.

9) https

:: http 웹프로토콜에 전송계층의 보안이 적용된 것이다. 이때의 전송계층보안 프로토콜은 TLS 이다.

10) DNS

::도메인 네임 시스템(Domain Name System, DNS)은 호스트의 도메인 이름을 호스트의 네트워크 주소로 바꾸거나 그 반대의 변환을 수행할 수 있도록 하기 위해 개발되었다.

특정 컴퓨터(또는 네트워크로 연결된 임의의 장치)의 주소를 찾기 위해, 사람이 이해하기 쉬운 도메인 이름을 숫자로 된 식별 번호(IP 주소)로 변환해 준다.
